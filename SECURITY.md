# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 1.1.1   | ✓ |
| < 1.1.1 | ✗ |

## Reporting a Vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Use GitHub's private vulnerability reporting instead:

1. Go to the [Security tab](../../security) of this repository.
2. Click **"Report a vulnerability"**.
3. Fill in the description, steps to reproduce, and potential impact.

Your report will be reviewed privately. We will not share details publicly until a fix is released.

## Response SLA

| Milestone | Target |
|-----------|--------|
| Acknowledgement | 7 days |
| Patch for Critical severity | 30 days |
| Patch for High severity | 60 days |

## Scope

This policy covers the `exposure_checker` Python package and the Gullwing desktop application distributed via GitHub Releases.
